### 1. 并发多流式对话（核心改动）

**文件**: `src/App.tsx`

**问题**: 用户在一个会话等待 AI 输出时，切换到另一个会话后旧的流被中止。

**方案**: 将全局单例 ref（`streamingRef` / `streamSessionRef` / `streamControllerRef`）替换为 `Map<string, StreamState>`。

```
// 旧架构（单例）
streamingRef          → boolean
streamSessionRef      → string | null
streamControllerRef   → AbortController | null
streamPersistTimerRef → Timer | null

// 新架构（Map）
activeStreamsRef = Map<sessionId, {
  controller: AbortController
  persistTimer: Timer | null
  sessionId: string
  prevMessages: ChatMessage[]
  fullContent: string
  fullThinking: string
}>
```

**关键行为变化**:
- `handleSelectSession` **不再 abort** 其他会话的流，只是切换视图
- 每个流有独立的 `prevMessages` / `fullContent` / `fullThinking` 闭包
- `setMessages` 更新时检查 `sessionIdRef.current === streamSessionId`，只更新当前显示的会话
- 每个流的持久化定时器独立运行，互不干扰
- 流完成/失败后只清理自己的 entry

**副作用**:
- 删除会话时若该会话正在流式，会 abort 并清理
- visibilitychange 保存全部活跃流
